Application

  • Bonne correspondance avec diagramme
  • Bugs et améliorations
    • Courriel d'un membre doit être le même que celui qu'il doit utiliser pour son compte
    • Quand un utilisateur se déconnecte, il peut entrer n'importe quel email ou mot de passe pour se reconnecter
    • Si un compte est en attente ou suspendu, signaler à l'utilisateur que son compte est en attente ou suspendu
    • Ne pas afficher les menus si le rôle d'un utilisateur n'est pas autorisé à exécuter les actions qui s'y trouvent
    • Quand il y a une erreur dans l'information d'entrée d'une activité ou d'un intérêt pour format non respecté, ne pas terminer le programme au moment de l'exception
      • Retourner au menu précédent lorsque cette situation arrive
    • Manque la notion d'unicité pour deux activités, intérêts ou membres ayant le même nom, la recherche par nom retournera le premier ayant ce nom, portant à confusion pour l'utilisateur
      • Possibilité d'empêcher la création de deux entités ayant le même nom
      • Bonne idée de lister les intérêts et les activités avec ID pour modification ou suppression, cette idée devrait être étendue à l'association des membres, des mots-clés et des intérêts à une activité
    • Imprimer le nom du membre responsable d'une activité, maintenant il imprime un identifiant (e.g., Membre@6bf2d08e)

Code

  • Classe MenuController un peu trop long, certaines des méthodes peuvent aller dans les contrôleurs pour les membres, les activités, les intérêts et les notifications
    • Mais on comprend l'idée que le Menu est utilisé pour créer les items dans le menu ainsi que les liens entre les menus
    • Peut utiliser un Enum indiquant le type du menu au lieu d'une liste de booleans pour déterminer quel menu c'est
  • Séparer les méthodes de vérification des champs comme méthodes à l'extérieur du constructeur de l'activité
  • Mauvaise gestion des string null, tente de vérifier la longueur du string même si le string est null
    • Possibilités:
      • Vérifier que le string n'est pas null avant de vérifier sa longueur
      • Remplacer string null par string vide
  • Mieux nommer les items du menu (e.g., utiliser menuMembres, menuArticles, menuOutils, etc. au lieu de menu1, menu2, menu3, etc.)
  • Pour faire les liens entre les menus, idéalement faire les setNextMenu avant de les ajouter dans le ArrayList des MenuItems pour éviter l'utilisation de get(index) pour identifier les items du menu, ce qui provoque un grand risque d'erreurs
  • Utiliser potentiellement les 'enhanced for loops' (e.g., for (Object o: collection)) au lieu d'itérer à travers les index
